# React 용어 사전 (Normal - 초중급)

> 알파벳 순서로 정리되어 있습니다. 한국어 번역과 함께 표기합니다.

---

## A

| 영어 | 한국어 | 설명 |
|------|--------|------|
| API (Application Programming Interface) | 응용 프로그래밍 인터페이스 | 소프트웨어 간 상호작용을 위한 규약 |
| Arrow Function | 화살표 함수 | `() => {}` 형태의 간결한 함수 표현식 |

## B

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Boolean | 불리언 | true 또는 false 값을 가지는 자료형 |
| Browser Router | 브라우저 라우터 | HTML5 History API를 사용하는 React Router의 라우터 컴포넌트 |
| Build | 빌드 | 소스 코드를 배포 가능한 형태로 변환하는 과정 |

## C

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Callback | 콜백 | 다른 함수에 인자로 전달되어 나중에 실행되는 함수 |
| Children | 자식 (children prop) | 컴포넌트 태그 사이에 포함된 콘텐츠를 전달하는 특별한 prop |
| Class Component | 클래스 컴포넌트 | ES6 class 문법을 사용하는 React 컴포넌트 (레거시) |
| Cleanup Function | 클린업 함수 | useEffect에서 반환하는 정리 함수, 이펙트 해제 시 실행됨 |
| Component | 컴포넌트 | UI를 구성하는 독립적이고 재사용 가능한 코드 단위 |
| Conditional Rendering | 조건부 렌더링 | 조건에 따라 다른 UI를 표시하는 기법 |
| Context | 컨텍스트 | props를 거치지 않고 컴포넌트 트리 전체에 데이터를 전달하는 방법 |
| Controlled Component | 제어 컴포넌트 | React state로 값이 관리되는 폼 요소 |
| CRUD | CRUD | Create(생성), Read(조회), Update(수정), Delete(삭제)의 약어 |

## D

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Dependency Array | 의존성 배열 | useEffect에서 이펙트 재실행 조건을 지정하는 배열 |
| Destructuring | 구조 분해 할당 | 객체나 배열의 값을 개별 변수로 추출하는 문법 |
| DOM (Document Object Model) | 문서 객체 모델 | HTML 문서의 구조를 트리 형태로 표현한 인터페이스 |
| Dynamic Route | 동적 경로 | URL 파라미터를 포함하는 라우트 (`/users/:id`) |

## E

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Event Handler | 이벤트 핸들러 | 사용자 인터랙션(클릭, 입력 등)에 반응하는 함수 |
| Expression | 표현식 | 값으로 평가되는 코드 조각, JSX 중괄호 안에 사용 |

## F

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Fragment | 프래그먼트 | 추가 DOM 노드 없이 여러 요소를 그룹화하는 `<></>` 문법 |
| Functional Component | 함수형 컴포넌트 | JavaScript 함수로 정의된 React 컴포넌트 |

## H

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Hook | 훅 | 함수형 컴포넌트에서 React 기능을 사용할 수 있게 하는 특별한 함수 |
| Hot Module Replacement (HMR) | 핫 모듈 교체 | 코드 변경 시 전체 페이지 새로고침 없이 모듈을 교체하는 기능 |

## I

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Immutability | 불변성 | 데이터를 직접 수정하지 않고 새로운 복사본을 만드는 원칙 |
| Import/Export | 가져오기/내보내기 | 모듈 간 코드를 공유하는 JavaScript 문법 |

## J

| 영어 | 한국어 | 설명 |
|------|--------|------|

## K

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Key | 키 | 리스트 렌더링 시 각 요소를 고유하게 식별하는 속성 |

## L

| 영어 | 한국어 | 설명 |
|------|--------|------|

| Lifting State Up | 상태 끌어올리기 | 공통 부모 컴포넌트로 상태를 이동시키는 패턴 |

## M

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Map | 맵 (배열 메서드) | 배열의 각 요소를 변환하여 새 배열을 반환하는 메서드 |
| Module | 모듈 | 독립적으로 관리되는 코드 파일 단위 |

## N

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Nested Route | 중첩 라우트 | 부모 라우트 안에 자식 라우트가 포함된 구조 |
| Node | 노드 | DOM 트리의 각 요소를 나타내는 단위 |
| npm | npm | Node.js 패키지 관리자 |

## O

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Outlet | 아웃렛 | React Router에서 중첩 라우트의 자식 컴포넌트가 렌더링되는 위치 |

## P

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Props | 프롭스 | 부모 컴포넌트에서 자식 컴포넌트로 전달되는 데이터 |
| Props Drilling | 프롭스 드릴링 | 여러 단계의 컴포넌트를 거쳐 props를 전달하는 문제 |
| Provider | 프로바이더 | Context 값을 하위 컴포넌트에 제공하는 컴포넌트 |

## R

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Reducer | 리듀서 | 이전 상태와 액션을 받아 새로운 상태를 반환하는 함수 |
| Ref | 레프 | DOM 요소나 값을 렌더링 없이 참조하는 Hook (useRef) |
| Re-render | 리렌더링 | 상태나 props 변경 시 컴포넌트가 다시 렌더링되는 것 |
| Route | 라우트 | URL 경로와 컴포넌트를 매핑하는 설정 |

## S

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Side Effect | 사이드 이펙트 | 컴포넌트 렌더링 외부에서 발생하는 작업 (API 호출, 타이머 등) |
| SPA (Single Page Application) | 단일 페이지 애플리케이션 | 하나의 HTML 페이지에서 동적으로 콘텐츠를 교체하는 웹 앱 |
| State | 상태 | 컴포넌트 내부에서 관리되는 동적 데이터 |
| Strict Mode | 엄격 모드 | 개발 중 잠재적 문제를 감지하는 React의 도구 |

## T

| 영어 | 한국어 | 설명 |
|------|--------|------|
| Template Literal | 템플릿 리터럴 | 백틱(`)으로 감싼 문자열, 변수 삽입 가능 |
| Ternary Operator | 삼항 연산자 | `조건 ? 참 : 거짓` 형태의 조건 표현식 |

## U

| 영어 | 한국어 | 설명 |
|------|--------|------|

| useEffect | useEffect | 사이드 이펙트를 수행하는 React Hook |

| useRef | useRef | 렌더링 없이 값을 참조하는 React Hook |
| useState | useState | 컴포넌트에 상태 변수를 추가하는 React Hook |

## V

|------|--------|------|

| Vite | Vite (비트) | 빠른 개발 서버와 빌드 도구를 제공하는 프론트엔드 빌드 도구 |
