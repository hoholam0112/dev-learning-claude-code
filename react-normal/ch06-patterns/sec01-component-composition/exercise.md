# 섹션 01: 컴포넌트 분리와 합성 - 연습 문제

## 개요

이 연습에서는 컴포넌트 설계의 핵심 개념들을 Node.js로 시뮬레이션합니다.
실제 React 컴포넌트 대신 순수 함수로 합성, 분리, 상태 끌어올리기 패턴을 연습합니다.

## 실행 방법

```bash
# 문제 풀기
node exercise.jsx

# 정답 확인
node solution.jsx
```

## 문제 목록

### 문제 1: 컴포넌트 분리 판단

- **난이도**: ★★☆☆☆
- **개념**: 단일 책임 원칙, 컴포넌트 분리 기준
- **설명**: 컴포넌트의 역할 수, 코드 줄 수, 재사용 가능성을 분석하여 분리 여부를 판단하는 함수를 작성합니다.
- **힌트**: 역할이 2개 이상이거나 코드가 50줄 초과이면 분리를 권장합니다. 단, 재사용 가능하더라도 역할이 하나이고 코드가 짧으면 분리가 불필요합니다.

### 문제 2: 상태 끌어올리기 시뮬레이션

- **난이도**: ★★★☆☆
- **개념**: Lifting State Up, 옵저버 패턴
- **설명**: 여러 자식 컴포넌트가 공유하는 상태를 부모에서 관리하는 패턴을 구현합니다. `subscribe`로 등록한 콜백이 값 변경 시 호출되어야 합니다.
- **힌트**: React에서 부모 상태가 변하면 자식이 리렌더링되는 것을 콜백 알림으로 시뮬레이션합니다.

### 문제 3: 레이아웃 합성 패턴

- **난이도**: ★★☆☆☆
- **개념**: children 패턴, 슬롯 패턴
- **설명**: header와 footer는 고정이고 중간 내용(children)은 동적으로 변하는 레이아웃 함수를 만듭니다.
- **힌트**: 결과 문자열에서 첫 줄은 `[Header: ...]`, 마지막 줄은 `[Footer: ...]` 형태입니다.
