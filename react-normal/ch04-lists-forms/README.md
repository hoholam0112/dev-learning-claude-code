# 챕터 04: 리스트와 폼

## 개요

React 애플리케이션에서 **리스트 렌더링**과 **폼 처리**는 가장 자주 사용하는 패턴입니다. 거의 모든 앱에는 목록(상품 목록, 댓글 목록, 할 일 목록 등)이 있고, 사용자 입력을 받는 폼(로그인, 검색, 회원가입 등)이 있습니다.

이 챕터에서는 배열 데이터를 UI로 변환하는 방법과, 사용자 입력을 React 방식으로 처리하는 방법을 학습합니다.

## 왜 중요한가?

- **map()과 key**: React에서 리스트를 렌더링할 때 `map()`을 사용하여 배열을 JSX 요소로 변환합니다. `key` prop은 React가 각 항목을 고유하게 식별하여 효율적으로 업데이트하기 위해 반드시 필요합니다. key가 없거나 잘못 지정되면 불필요한 리렌더링이 발생하고 버그가 생길 수 있습니다.
- **폼 처리**: React에서는 HTML 폼 요소의 값을 state로 관리하는 **제어 컴포넌트(Controlled Component)** 패턴을 사용합니다. 이를 통해 입력값의 유효성 검사, 조건부 비활성화, 실시간 피드백 등을 쉽게 구현할 수 있습니다.

## 포함된 섹션

| 섹션 | 제목 | 설명 |
|------|------|------|
| sec01-list-rendering | 리스트 렌더링 | map()으로 리스트 렌더링, key prop의 중요성 |
| sec02-form-handling | 폼 처리 | 제어 컴포넌트, 입력 처리, 유효성 검사 |

## 학습 순서

1. **sec01-list-rendering** → 배열 데이터를 리스트로 렌더링하는 방법을 학습합니다
2. **sec02-form-handling** → 사용자 입력을 React 방식으로 처리하는 방법을 학습합니다

## 선수 지식

- JavaScript 배열 메서드 (map, filter, reduce)
- React 기초 (JSX, 컴포넌트, props, state)
- 이벤트 핸들링 기초
