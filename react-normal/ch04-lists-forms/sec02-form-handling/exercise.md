## 문제 1: 폼 상태 관리 객체 만들기 (⭐⭐)

### 설명
React의 제어 컴포넌트 패턴을 시뮬레이션하는 폼 상태 관리 객체를 만드세요. 필드 이름 배열을 받아 각 필드의 값을 읽고 쓸 수 있는 객체를 반환합니다.

### 요구 사항
- 필드 이름 배열을 받아 상태 객체를 생성합니다
- `getValue(fieldName)`으로 현재 값을 읽을 수 있습니다
- `setValue(fieldName, value)`로 값을 설정할 수 있습니다
- `getAll()`로 모든 필드의 현재 값을 객체로 반환합니다
- `reset()`으로 모든 필드를 빈 문자열로 초기화합니다

## 문제 2: 폼 유효성 검사 (⭐⭐)

### 설명
폼 데이터와 유효성 검사 규칙을 받아 검증 결과를 반환하는 함수를 작성하세요. React에서 폼 제출 시 유효성 검사를 수행하는 패턴을 연습합니다.

### 요구 사항
- 데이터 객체와 규칙 객체를 받습니다
- 지원하는 규칙: `required`(필수), `minLength`(최소 길이), `pattern`(정규식)
- 반환값: `{ isValid: boolean, errors: { 필드명: "오류 메시지", ... } }`
- 여러 규칙이 있으면 첫 번째 실패한 규칙의 메시지만 반환합니다

---

## 문제 3: 폼 제출 처리 (⭐⭐)

### 설명
유효성 검사와 제출을 결합하는 함수를 작성하세요. 실제 React 폼의 `onSubmit` 핸들러와 유사한 흐름을 구현합니다.

### 요구 사항
- 폼 데이터와 유효성 검사 규칙을 받습니다
- 검증에 성공하면 `{ success: true, data: 폼데이터 }`를 반환합니다
- 검증에 실패하면 `{ success: false, errors: 오류객체 }`를 반환합니다
