## 문제 1: 프로필 수정 폼 (⭐⭐)

### 설명
사용자 프로필을 수정하는 **제어 컴포넌트** 기반 폼을 구현하세요. 기존 프로필 데이터를 초기값으로 표시하고, 수정 후 저장할 수 있어야 합니다.

### 요구 사항
- 다음 필드를 포함할 것: 닉네임(텍스트), 자기소개(textarea), 관심 분야(select - 프론트엔드/백엔드/풀스택), 뉴스레터 수신 동의(checkbox)
- 초기값이 설정되어 있어야 함 (예: 닉네임 "리액트러버", 자기소개 "안녕하세요!", 관심 분야 "프론트엔드", 뉴스레터 수신 true)
- 하나의 `handleChange` 함수로 모든 필드 변경을 처리할 것
- "저장" 버튼 클릭 시 현재 폼 데이터를 `console.log`로 출력
- "초기화" 버튼 클릭 시 초기값으로 복원

### 힌트
<details><summary>힌트 보기</summary>

- checkbox는 `e.target.checked`로 값을 가져옵니다
- `e.target.type`으로 checkbox인지 판단할 수 있습니다
- select의 value는 일반 텍스트 input과 동일하게 처리합니다
- 초기값 객체를 별도 상수로 분리하면 초기화가 쉬워집니다

</details>

## 문제 2: 실시간 검색 필터 (⭐⭐)

### 설명
과일 목록에서 입력한 키워드로 실시간 필터링하는 **검색 컴포넌트**를 구현하세요. `useRef`를 활용하여 페이지 로드 시 검색 입력란에 자동으로 포커스를 맞추세요.

### 요구 사항
- 과일 목록 데이터: `['사과', '바나나', '포도', '딸기', '수박', '참외', '복숭아', '키위', '망고', '블루베리']`
- 검색어를 입력하면 실시간으로 목록이 필터링되어야 함 (제어 컴포넌트 사용)
- `useRef`를 사용하여 컴포넌트 마운트 시 입력란에 자동 포커스
- 검색 결과 개수를 표시할 것 (예: "3개의 결과")
- "초기화" 버튼 클릭 시 검색어 삭제 및 입력란에 포커스
- 검색 결과가 없을 때 "검색 결과가 없습니다" 메시지 표시

### 힌트
<details><summary>힌트 보기</summary>

- `useEffect`와 `useRef`를 함께 사용하여 마운트 시 포커스를 줄 수 있습니다:
  ```jsx
  useEffect(() => { inputRef.current.focus(); }, []);
  ```
- `Array.filter()`와 `String.includes()`를 조합하면 검색 필터를 구현할 수 있습니다
- 대소문자 구분 없이 검색하려면 `toLowerCase()`를 사용하세요 (한국어에는 불필요하지만 연습용)

</details>

## 문제 3: 다단계 설문조사 폼 (⭐⭐⭐)

### 설명
3단계로 구성된 설문조사 폼을 구현하세요. 각 단계에서 입력한 데이터는 유지되어야 하며, 마지막 단계에서 전체 데이터를 확인하고 제출할 수 있습니다.

### 요구 사항
- **1단계**: 이름(텍스트), 나이(숫자 입력)
- **2단계**: 좋아하는 프로그래밍 언어(select - JavaScript/Python/Java/기타), 경력(라디오 - 학생/1~3년/3년 이상)
- **3단계**: 입력한 모든 데이터를 요약 표시 + "수정하기"(이전 단계로) / "제출하기" 버튼
- 현재 진행 단계를 시각적으로 표시 (예: "1단계 / 3단계", 또는 프로그레스 바)
- "이전" / "다음" 버튼으로 단계 이동
- 각 단계에서 필수 필드가 비어있으면 다음으로 넘어갈 수 없도록 유효성 검사
- 제출 후 "제출 완료" 화면 표시

### 힌트
<details><summary>힌트 보기</summary>

- `currentStep` state로 현재 단계를 관리합니다
- 모든 단계의 데이터를 하나의 state 객체에 저장하면 단계 이동 시에도 데이터가 유지됩니다
- 라디오 버튼은 같은 `name`을 가진 input 그룹으로 구현합니다:
  ```jsx
  <input type="radio" name="experience" value="student"
    checked={formData.experience === 'student'}
    onChange={handleChange} />
  ```
- 조건부 렌더링으로 현재 단계에 해당하는 폼만 표시합니다

</details>
