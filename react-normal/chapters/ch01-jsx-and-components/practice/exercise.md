## 문제 1: 자기소개 카드 만들기 (난이도: ⭐)

### 설명
본인의 정보를 표시하는 자기소개 카드 컴포넌트를 만들어보세요. JSX의 기본 문법과 JavaScript 표현식 삽입을 연습합니다.

### 요구 사항
- `IntroCard`라는 함수형 컴포넌트를 만드세요
- 다음 정보를 변수로 선언하고 JSX에서 표현식으로 사용하세요:
  - 이름 (문자열)
  - 나이 (숫자)
  - 취미 (배열, 최소 3개)
  - 자기소개 한 줄 (문자열)
- 취미 배열은 `join(", ")`을 사용하여 하나의 문자열로 표시하세요
- `className`을 사용하여 CSS 클래스를 지정하세요 (실제 CSS 작성은 선택 사항)
- 프로필 이미지를 포함하세요 (`<img />` 태그, placeholder URL 사용 가능)

### 힌트
<details><summary>힌트 보기</summary>

- 컴포넌트 이름은 대문자로 시작해야 합니다
- `<img />` 태그는 반드시 self-closing으로 작성하세요
- 배열을 문자열로 변환: `{hobbies.join(", ")}`
- 여러 줄의 JSX는 소괄호 `()`로 감싸세요

</details>

## 문제 2: 레스토랑 메뉴 페이지 (난이도: ⭐)

### 설명
레스토랑의 메뉴를 보여주는 간단한 페이지를 컴포넌트로 구성해보세요. 컴포넌트 분리와 중첩을 연습합니다.

### 요구 사항
- 다음 컴포넌트들을 만드세요:
  1. `RestaurantHeader` - 레스토랑 이름과 설명
  2. `MenuItem` - 메뉴 하나 (이름, 가격, 설명)
  3. `MenuSection` - 여러 `MenuItem`을 포함하는 섹션
  4. `RestaurantFooter` - 영업시간, 연락처
  5. `RestaurantApp` - 위 모든 컴포넌트를 조합하는 루트 컴포넌트
- `RestaurantApp`에서 Fragment(`<>...</>`)를 사용하세요
- 각 `MenuItem`에 가격은 숫자 변수로 선언하고, `{price.toLocaleString()}원` 형태로 표시하세요

### 힌트
<details><summary>힌트 보기</summary>

- Fragment 문법: `<>내용</>`
- 숫자 포맷팅: `{(12000).toLocaleString()}` => `"12,000"`
- 아직 props를 배우지 않았으므로 각 `MenuItem` 내부에 데이터를 하드코딩해도 괜찮습니다
- 컴포넌트를 먼저 작은 단위부터 만들고, 큰 단위에서 조합하세요

</details>

## 문제 3: 날씨 대시보드 (난이도: ⭐⭐)

### 설명
현재 날씨 정보를 표시하는 대시보드를 여러 컴포넌트로 구성해보세요. JSX 표현식, 컴포넌트 분리, 스타일링을 종합적으로 연습합니다.

### 요구 사항
- 다음 컴포넌트들을 만드세요:
  1. `WeatherIcon` - 날씨 상태에 따른 이모지 표시 (예: 맑음 = "☀️", 흐림 = "☁️")
  2. `TemperatureDisplay` - 현재 온도를 표시 (변수 사용, 섭씨/화씨 모두 표시)
  3. `WeatherDetails` - 습도, 풍속, 강수확률 등 세부 정보
  4. `WeatherDashboard` - 위 컴포넌트들을 조합한 대시보드
- 섭씨를 화씨로 변환하는 계산을 JSX 표현식 안에서 수행하세요
  - 공식: `(celsius * 9/5) + 32`
- 인라인 스타일을 사용하여 온도에 따라 다른 색상을 적용하세요
  - 30도 이상: 빨간색, 20~29도: 주황색, 20도 미만: 파란색
- 컴포넌트 트리 구조를 주석으로 작성하세요

### 힌트
<details><summary>힌트 보기</summary>

- 삼항 연산자 중첩: `condition1 ? value1 : condition2 ? value2 : value3`
- 인라인 스타일의 동적 값: `style={{ color: temp >= 30 ? 'red' : 'blue' }}`
- 소수점 처리: `number.toFixed(1)`으로 소수 첫째 자리까지 표시
- 컴포넌트 트리는 주석으로 `/*  */` 안에 작성하면 됩니다

</details>
